# 回顾标准

> **语言**: [English](../../../core/retrospective-standards.md) | 简体中文

**版本**: 1.0.0
**最后更新**: 2026-03-26
**适用范围**: 所有实践迭代开发的软件团队
**范畴**: universal

---

## 目的

回顾是结构化的团队反思，识别做得好的、需要改善的，以及具体的改善行动。它们是敏捷方法论的关键实践。

---

## 引导技法

| 技法 | 适用场景 | 结构 |
|------|---------|------|
| **Start-Stop-Continue** | 快速、默认 | 开始 / 停止 / 继续 |
| **Starfish** | 更细致分析 | 更多 / 保持 / 减少 / 开始 / 停止 |
| **4Ls** | 团队建设 | 喜欢 / 学到 / 缺少 / 渴望 |
| **Sailboat** | 可视化 | 风 / 锚 / 礁石 / 岛屿 |

---

## 行动项目追踪规则

1. 每次回顾开始先检视未完成项目。
2. 超过 2 个循环未完成须重新评估。
3. 每次回顾最多 3 个行动项目。
4. 每项须有单一负责人。

---

## 最佳实践

1. **定期举行** — 一致性建立信任。
2. **变化技法** — 轮替使用。
3. **以数据开始** — 用指标引导讨论。
4. **以承诺结尾** — 每次产出 1-3 个行动项目。
5. **追踪落实** — 这是价值产生的地方。
