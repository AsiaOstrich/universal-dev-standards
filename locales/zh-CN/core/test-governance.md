# 测试治理标准

**适用范围**: 所有软件项目
**范围**: universal

---

## 概述

测试治理标准定义测试活动的策略、完成条件和环境管理策略。本标准将 ISO/IEC/IEEE 29119 正式过程与敏捷/Scrum 实践相结合。

## 参考

| 标准/来源 | 内容 |
|-----------|------|
| ISO/IEC/IEEE 29119-2 | 测试过程 |
| ISO/IEC/IEEE 29119-3 | 测试文档（测试计划/测试用例） |
| ISO/IEC/IEEE 12207 | 验证过程 + 集成过程 |
| 敏捷/Scrum 指南 | 完成定义（DoD） |
| ISTQB 基础教学大纲 | 测试级别、测试类型 |
| Mike Cohn 测试金字塔 | 金字塔经验比例（推荐默认值） |

## 术语

| 术语 | 标准来源 | 描述 |
|------|---------|------|
| 完成定义（DoD） | 敏捷/Scrum | 任务/功能完成的检查清单 |
| 测试完成条件 | ISO 29119-2 | 测试活动的退出条件 |
| 系统集成测试 | ISO 12207 | 非正式 ISO 术语，但为行业标准 |
| 金字塔比例 70/20/7/3 | Mike Cohn（经验值） | 推荐默认值，非强制 |

## 测试策略

### 品质目标

| ID | 名称 | 度量 |
|----|------|------|
| QO-1 | 功能正确性 | 测试通过率 >= 95% |
| QO-2 | 回归预防 | CI 中零回归失败 |
| QO-3 | 代码品质 | 零 lint/类型错误，静态分析通过 |

### 级别归属

| 级别 | 负责人 | 环境 | 金字塔比例 |
|------|--------|------|-----------|
| 单元测试（UT） | 开发者 | 本地 | 70% |
| 集成测试（IT） | 开发者 | 本地 / CI | 20% |
| 系统测试（ST） | QA / 开发者 | CI / SIT | 7% |
| E2E 测试 | QA | 预发布 | 3% |

### 静态分析

在测试执行前运行静态分析工具：

- `ruff check .`（Python）
- `eslint .`（TypeScript/JavaScript）
- `cargo clippy`（Rust）

## 测试完成条件

### 按级别定义

| 级别 | 完成条件 |
|------|---------|
| **单元测试** | 所有测试通过、覆盖率 >= 目标、无 TODO 测试 |
| **集成测试** | API 契约验证通过、数据库迁移测试通过 |
| **系统测试** | 所有用户场景通过、性能目标达标 |
| **E2E 测试** | 关键用户流程通过、无阻塞级缺陷 |

### 完成定义（DoD）检查清单

- [ ] 所有测试通过
- [ ] 代码审查已完成
- [ ] 文档已更新
- [ ] 无已知阻塞级缺陷
- [ ] 性能目标达标
- [ ] 安全扫描通过

## 环境管理

| 环境 | 用途 | 数据 |
|------|------|------|
| **本地** | 开发和单元测试 | 模拟数据 |
| **CI** | 自动化测试 | 测试夹具 |
| **SIT** | 集成测试 | 模拟数据 |
| **预发布** | E2E 和验收测试 | 脱敏生产数据 |

## 规则

| ID | 触发条件 | 指令 | 优先级 |
|----|---------|------|--------|
| enforce-completion-criteria | 完成任务或功能时 | 在将任务/功能标记为完成之前，验证所有必要的完成条件已满足 | 必须 |
| pyramid-compliance | 规划测试策略时 | 以 70/20/7/3 金字塔比例作为指导方针。允许偏差，但需有文档记录的理由 | 必须 |
| sit-isolation | 运行系统测试时 | 系统测试应对外部依赖使用 Stub，但使用真实的内部服务。使用 SIT 环境进行系统级验证 | 建议 |
| test-execution-continuity | 新增或完成测试用例时 | 测试用例必须连接到自动化执行触发器（CI gate、build hook 或定时执行）。存在但从未执行的测试提供假信心，比没有测试更糟。在将测试覆盖率标记为完成前，请确认执行历程存在。| 必须 |
