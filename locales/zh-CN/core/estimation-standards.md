# 估算标准

> 版本: 1.0.0 | 最后更新: 2026-04-01

## 概述

本文件定义软件开发项目的估算标准，涵盖估算方法、校准机制、常见反模式、信心等级、重新估算触发条件，以及估算与承诺的关键区别。

好的估算不在于精确，而在于对不确定性保持透明，并通过校准持续改进。

## 估算方法

### 1. Planning Poker

共识式估算技术，团队成员独立使用编号卡片估算，然后讨论收敛。

| 方面 | 说明 |
|------|------|
| **适用场景** | Sprint 规划、Story 级别估算 |
| **团队规模** | 3–9 人 |
| **尺度** | Fibonacci（1, 2, 3, 5, 8, 13, 21） |

### 2. T-Shirt Sizing

使用抽象尺寸标签（XS, S, M, L, XL）的相对大小估算技术。

| 方面 | 说明 |
|------|------|
| **适用场景** | Roadmap 规划、Epic 级别估算 |
| **尺度** | XS, S, M, L, XL |

### 3. 三点估算

使用乐观值（O）、最可能值（M）、悲观值（P）的统计技术。

| 方面 | 说明 |
|------|------|
| **适用场景** | 关键路径项目、高不确定性任务 |
| **公式** | E = (O + 4M + P) / 6; SD = (P - O) / 6 |

## 校准

团队应定期将估算与实际结果比对，以持续提升准确度。

**估算准确率**：`实际工作量 / 估算工作量`
- 比率 = 1.0：完美估算
- 比率 > 1.0：低估（实际花更久）
- 比率 < 1.0：高估（实际花更少）

## 反模式

| 反模式 | 说明 | 缓解策略 |
|--------|------|---------|
| **锚定偏误** | 第一个数字主导后续估算 | 使用盲估（如同时翻牌） |
| **规划谬误** | 系统性低估时间和风险 | 使用历史校准数据 |
| **范围蔓延盲区** | 未注意逐渐扩大的范围 | 估算前锁定范围定义 |
| **学生综合症** | 拖到最后才开始工作 | 设定中间里程碑 |
| **帕金森定律** | 工作扩展以填满可用时间 | 使用相对估算（Story Points） |

## 信心等级

| 等级 | 变异范围 | 适用场景 |
|------|---------|---------|
| **高** | ±20% | 需求明确、有先前经验的熟悉工作 |
| **中** | ±50% | 部分已知、有些未知的工作 |
| **低** | ±100% | 高度不确定、新技术或不明确需求 |

## 重新估算触发条件

| 触发条件 | 行动 |
|---------|------|
| **需求变更** | 重新估算受影响项目 |
| **技术发现** | 以更新的知识重新估算 |
| **外部依赖变更** | 重新估算集成工作 |
| **时间阈值超过** | 进行检查点审查 |

## 估算 vs 承诺

| 方面 | 估算 | 承诺 |
|------|------|------|
| **本质** | 带有不确定性的预测 | 带有责任的承诺 |
| **负责人** | 开发团队 | 管理层 / Product Owner |
| **格式** | 范围 + 信心等级 | 具体日期或范围 |
| **变更** | 随信息更新而调整 | 与利益相关方重新协商 |
