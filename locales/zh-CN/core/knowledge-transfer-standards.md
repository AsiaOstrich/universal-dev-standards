# 知识传承标准

> 版本: 1.0.0 | 最后更新: 2026-04-01

## 概述

本文件定义知识传承标准，涵盖 Onboarding、Handoff、Bus Factor 评估及 Code Tour 机制。目标是确保关键知识不会因团队成员离职或角色转换而消失。

---

## Onboarding 路线图

### 4 周计划

| 周次 | 主题 | 目标 | 交付物 |
|------|------|------|--------|
| **第 1 周** | 环境与文化 | 完成开发环境设置；理解团队规范 | 成功构建并运行项目 |
| **第 2 周** | 架构与代码 | 理解系统架构与关键路径 | 完成 Code Tour |
| **第 3 周** | 流程与工具 | 理解 CI/CD、部署与监控流程 | 完成第一个 PR |
| **第 4 周** | 独立贡献 | 独立完成一个小型任务 | 通过 Buddy 评估 |

### Onboarding 检查清单

| 类别 | 项目 |
|------|------|
| **账号访问** | Git 仓库、CI/CD 平台、监控工具、云端控制台、通讯工具 |
| **环境设置** | 开发环境、IDE 配置、VPN、数据库访问、本地构建验证 |
| **文档** | README、架构文档、API 文档、CLAUDE.md 或等效 AI 上下文文件 |
| **会议** | 与 Buddy 1:1、团队介绍、架构概览 |
| **首要任务** | Good First Issue、Code Tour、第一个小型 PR |

## Handoff 流程

| 项目 | 说明 | 时程 |
|------|------|------|
| **负责清单** | 列出所有负责的模块、服务与流程 | 离职前 4 周 |
| **隐性知识记录** | 记录所有「只有我知道」的操作与陷阱 | 离职前 3 周 |
| **WIP 交接** | 移交进行中的任务、打开的 PR、事件追踪 | 离职前 2 周 |
| **Buddy 配对** | 指定继任者并进行 Pair Programming | 离职前 2 周 |
| **账号与权限** | 移交 owner 权限、轮换 API 密钥 | 离职前 1 周 |
| **Runbook 更新** | 审查并更新所有相关 Runbook | 离职前 1 周 |

### 知识记录格式

| 字段 | 说明 |
|------|------|
| **主题** | 捕捉什么知识 |
| **情境** | 何时需要此知识 |
| **步骤** | 具体操作程序 |
| **陷阱** | 常见错误与注意事项 |
| **资源** | 代码、文档、Runbook 链接 |

## Bus Factor 评估

| 指标 | 定义 | 风险等级 |
|------|------|---------|
| **知识人数** | 可独立处理该领域的人数 | 1 人 = **高**、2 人 = **中**、3+ 人 = **低** |
| **文档覆盖率** | 有书面文档覆盖的百分比 | < 30% = **高**、30-70% = **中**、> 70% = **低** |
| **最近知识分享** | 最近一次有人学习该领域 | > 6 个月 = **高**、3-6 个月 = **中**、< 3 个月 = **低** |

### 知识扩散策略

| 策略 | 适用场景 |
|------|---------|
| **Pair Programming** | 复杂逻辑、特殊技巧 |
| **Tech Talk** | 架构决策、设计理由 |
| **文档化** | 操作程序、部署步骤 |
| **Code Review 轮调** | 代码理解、打破知识孤岛 |
| **On-call 轮调** | 运维知识、事件响应 |

## Code Tour

### 导览路线

| 路线 | 涵盖内容 | 目标对象 |
|------|---------|---------|
| **Quick Start** | 入口点、主要配置、启动流程 | 所有新人 |
| **Request Flow** | 请求从进入到响应的完整路径 | 后端开发者 |
| **Data Flow** | 数据从创建到存储到检索的路径 | 数据相关开发者 |
| **Deploy Flow** | 从 PR 到生产的完整路径 | 需要部署的人 |
| **Key Decisions** | 重要架构决策及其 ADR | 需要理解「为什么」的人 |
