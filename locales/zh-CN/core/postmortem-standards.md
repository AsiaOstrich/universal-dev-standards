# Postmortem 标准（无责事后检讨）

> 版本: 1.0.0 | 最后更新: 2026-04-01

## 概述

本文件定义生产环境事件后进行无责 Postmortem 的标准。Postmortem 是一个结构化流程，用于分析发生了什么、为什么发生、以及如何防止再次发生 — 不归咎于个人。

事件响应工作流程（检测 → 分级 → 缓解 → 解决）请参阅 [Incident Response Assistant](../../../skills/incident-response-assistant/SKILL.md)。

---

## 触发条件

### 强制执行

| 条件 | 理由 |
|------|------|
| **SEV-1 事件**（一律） | 任何完全服务中断需要彻底分析 |
| **SEV-2 事件持续 > 1 小时** | 延长的部分中断指出系统性问题 |
| **任何涉及数据丢失的事件** | 数据丢失具有不可逆后果 |
| **任何造成 SLA 违反的事件** | SLA 违规有合同影响 |
| **任何涉及安全漏洞的事件** | 安全事件需要详细取证分析 |

### 建议执行

| 条件 | 理由 |
|------|------|
| SEV-2 事件持续 < 1 小时 | 短暂但重大；可能揭示系统性风险 |
| SEV-3 重复事件（30 天内 2 次以上，相同根因） | 重复表示未解决的系统性问题 |
| 具有大潜在影响的 Near-miss | 无实际损害的学习机会 |

## 无责原则

| 原则 | 说明 |
|------|------|
| **人会犯错** | 错误是复杂系统的自然产物，不是个人缺陷 |
| **聚焦系统** | 分析系统如何允许错误发生，而非谁犯了错 |
| **心理安全** | 参与者可以公开分享，不必担心惩罚 |
| **学习导向** | 目标是学习和改善，不是追究责任 |

### 语言指南

| 正确（系统聚焦） | 错误（归咎聚焦） |
|-----------------|-----------------|
| 「部署包含未覆盖的边界案例」 | 「工程师 A 部署了有 Bug 的代码」 |
| 「监控系统未在 5 分钟内检测到异常」 | 「值班工程师忽略了告警」 |

---

## 根因分析方法

| 方法 | 最适用场景 |
|------|-----------|
| **5 Whys** | 单一事件链，因果关系清楚 |
| **Ishikawa（鱼骨图）** | 多个可能原因，需要分类 |
| **Fault Tree Analysis** | 安全关键系统、概率分析 |
| **Timeline Analysis** | 涉及多团队、多系统的复杂事件 |
| **Change Analysis** | 在近期变更后发生的事件 |

### Root Cause vs Trigger vs Contributing Factor

| 类型 | 定义 | 示例 |
|------|------|------|
| **Root Cause** | 若移除，事件不会发生 | CI/CD 缺乏 SQL 性能分析 |
| **Trigger** | 直接启动事件的事件 | 新功能部署 |
| **Contributing Factor** | 加重影响但非根因 | 监控告警延迟 10 分钟 |

## Quick Reference Card

### 是否需要编写 Postmortem？
```
SEV-1？                   → 是（一律）
SEV-2 且 > 1 小时？       → 是
数据丢失？                → 是
SLA 违反？                → 是
安全漏洞？                → 是
SEV-2 且 < 1 小时？       → 建议
SEV-3 重复发生？          → 建议（简化版）
SEV-3/4 首次发生？        → 否（记录于事件日志）
```

### Postmortem 时间线
```
事件解决 → 24h → 收集事实
         → 48h → 编写草稿
         → +3 个工作日 → 召开会议
         → +24h → 定稿并发布
         → 持续 → 跟踪行动项目
```

# 事后检讨标准（Blameless Postmortem）

> 版本: 1.0.0 | 最后更新: 2026-04-01

## 概述

本文件定义无指责事后检讨的流程，涵盖触发条件、Blameless 原则、根因分析方法论和行动项目追踪。

## 触发条件

| 条件 | 是否强制 |
|------|---------|
| SEV-1 事故 | 强制 |
| SEV-2 > 1 小时 | 强制 |
| 数据丢失 | 强制 |
| SLA 违反 | 强制 |
