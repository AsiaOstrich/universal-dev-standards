# 无障碍标准

**版本**: 1.0.0
**最后更新**: 2026-01-29
**适用范围**: 所有包含用户界面的软件项目
**范围**: universal
**行业标准**: WCAG 2.1/2.2, WAI-ARIA 1.2, Section 508
**参考**: [w3.org](https://www.w3.org/WAI/WCAG21/quickref/)

---

## 目的

本标准定义创建无障碍软件的综合指南，使具有不同能力的人都能使用。标准与国际规范对齐，包括 WCAG 2.1，并支持全球无障碍法规的合规性。

**参考标准**:
- [WCAG 2.1](https://www.w3.org/WAI/WCAG21/quickref/) - 网页内容无障碍指南
- [WCAG 2.2](https://www.w3.org/TR/WCAG22/) - 最新 WCAG 版本（2023）
- [WAI-ARIA 1.2](https://www.w3.org/TR/wai-aria-1.2/) - 无障碍富互联网应用
- [Section 508](https://www.section508.gov/) - 美国联邦无障碍要求

## 无障碍原则（POUR）

```
┌─────────────────────────────────────────────────────────────────┐
│               WCAG 2.1 POUR 原则                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   P - 可感知（PERCEIVABLE）                                      │
│   ├── 用户必须能够感知信息                                        │
│   ├── 为非文本内容提供文本替代                                    │
│   ├── 为多媒体提供字幕和替代方案                                  │
│   └── 内容可适配且可区分                                          │
│                                                                 │
│   O - 可操作（OPERABLE）                                         │
│   ├── 所有功能可通过键盘访问                                      │
│   ├── 用户有足够时间阅读和使用内容                                │
│   ├── 不要使用可能引发癫痫的设计                                  │
│   └── 帮助用户导航和查找内容                                      │
│                                                                 │
│   U - 可理解（UNDERSTANDABLE）                                   │
│   ├── 文本可读且可理解                                            │
│   ├── 网页以可预测的方式呈现和操作                                │
│   └── 帮助用户避免和纠正错误                                      │
│                                                                 │
│   R - 健壮（ROBUST）                                             │
│   ├── 与当前和未来的用户工具兼容                                  │
│   └── 使用标准化标记语言                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## WCAG 合规级别

| 级别 | 描述 | 目标 |
|------|------|------|
| **A** | 最低级别 — 基本无障碍 | 所有项目的最低要求 |
| **AA** | 中级 — 解决最常见的障碍 | **推荐目标**（大多数法规要求） |
| **AAA** | 最高级别 — 最全面的无障碍 | 针对特定需求的增强 |

## 键盘导航

| 要求 | 说明 |
|------|------|
| 所有交互元素可通过 Tab 键聚焦 | 按钮、链接、表单控件 |
| 焦点顺序符合逻辑 | 遵循视觉布局顺序 |
| 焦点指示器清晰可见 | 不要移除默认焦点样式 |
| 提供跳过导航链接 | 允许用户跳过重复内容 |
| 不要创建键盘陷阱 | 用户可以自由导航进出所有组件 |

## 颜色与对比度

| 类型 | AA 级别 | AAA 级别 |
|------|---------|----------|
| 普通文本 | 4.5:1 | 7:1 |
| 大文本（18px+） | 3:1 | 4.5:1 |
| UI 组件和图标 | 3:1 | 不适用 |

## 无障碍检查清单

- [ ] 所有图片都有 `alt` 属性
- [ ] 所有表单控件都有关联的标签
- [ ] 页面有正确的标题层级结构
- [ ] 颜色不是传达信息的唯一手段
- [ ] 对比度符合 AA 级别要求
- [ ] 所有功能都可通过键盘操作
- [ ] 使用 ARIA 属性增强语义
- [ ] 视频有字幕，音频有文字记录

## 测试与验证

| 工具 | 类型 | 用途 |
|------|------|------|
| axe | 自动化 | 通用无障碍测试 |
