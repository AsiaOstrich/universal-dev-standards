# 程式码审查检查清单

> **语言**: [English](../../../../../skills/claude-code/code-review-assistant/review-checklist.md) | 繁体中文

**版本**: 1.0.0
**最后更新**: 2025-12-24
**适用范围**: Claude Code Skills

---

## 目的

本文件提供全面的程式码审查检查清单，以确保程式码品质和标准合规性。

---

## 1. 功能性

- [ ] **程式码执行预期功能**
  - 已验证需求/规格对齐
  - 满足验收标准
  - 处理边界情况

- [ ] **无明显错误**
  - 存在 null/undefined 检查
  - 检查阵列边界
  - 处理错误条件

- [ ] **逻辑正确**
  - 条件合理
  - 回圈正确终止
  - 计算准确

---

## 2. 设计与架构

- [ ] **遵循专案架构**
  - 尊重分层
  - 维持关注点分离
  - 依赖方向正确

- [ ] **使用适当的设计模式**
  - 无过度设计
  - 无设计不足
  - 正确应用模式

- [ ] **程式码放置在正确位置**
  - 档案逻辑组织
  - 相关程式码集中放置

---

## 3. 程式码品质

- [ ] **遵循编码标准**
  - 遵守命名惯例
  - 格式一致
  - 遵循样式指南

- [ ] **无程式码异味**
  - 方法 ≤50 行
  - 类别具有单一职责
  - 圈复杂度 ≤10
  - 无深层巢状条件（≤3 层）

- [ ] **应用 DRY 原则**
  - 无重复程式码区块
  - 提取共用逻辑

---

## 4. 可读性

- [ ] **程式码易于理解**
  - 变数名称具描述性
  - 函式名称揭示意图
  - 逻辑自然流畅

- [ ] **注解有帮助**
  - 解释复杂逻辑
  - 记录为什么（WHY），而非什么（WHAT）
  - 无被注解掉的程式码

---

## 5. 测试

- [ ] **存在测试**
  - 新程式码有测试
  - 测试涵盖正常路径
  - 测试涵盖错误情况
  - 测试边界情况

- [ ] **测试品质良好**
  - 测试可读
  - 测试名称描述场景
  - 断言清晰
  - 无不稳定测试

- [ ] **维持测试覆盖率**
  - 覆盖率未降低
  - 涵盖关键路径

---

## 6. 安全性

- [ ] **无安全漏洞**
  - 无 SQL 注入风险
  - 无 XSS 漏洞
  - 无硬编码密钥
  - 无不安全的依赖

- [ ] **存在输入验证**
  - 清理使用者输入
  - 执行型别检查
  - 强制执行大小限制

- [ ] **身份验证/授权正确**
  - 适当的身份验证检查
  - 强制执行基于角色的存取
  - 保护敏感资料

---

## 7. 效能

- [ ] **无明显效能问题**
  - 无 N+1 查询
  - 无不必要的回圈
  - 热路径中无阻塞操作

- [ ] **使用高效演算法**
  - 考虑复杂度
  - 适当的资料结构
  - 有益处时使用快取

- [ ] **资源管理适当**
  - 关闭连线
  - 防止记忆体泄漏
  - 释放档案控制代码

---

## 8. 错误处理

- [ ] **适当处理错误**
  - 存在 try-catch 区块
  - 捕获特定例外
  - 避免通用捕获

- [ ] **错误讯息有帮助**
  - 讯息具描述性
  - 包含可操作资讯
  - 无暴露敏感资料

- [ ] **日志记录充分**
  - 记录错误与上下文
  - 日志层级适当
  - 无过度日志记录

---

## 9. 文件

- [ ] **存在 API 文件**
  - 记录公开方法
  - 解释参数
  - 描述回传值
  - 记录例外

- [ ] **如需要则更新 README**
  - 记录新功能
  - 更新安装说明

- [ ] **更新 CHANGELOG（如适用）**
  - 将面向使用者的变更加入 `[Unreleased]`
  - 以 **BREAKING** 标记破坏性变更

---

## 10. 依赖

- [ ] **依赖有正当理由**
  - 新依赖必要
  - 授权相容
  - 无安全漏洞
  - 积极维护

- [ ] **依赖版本已锁定**
  - 指定确切版本
  - 更新锁定档案
