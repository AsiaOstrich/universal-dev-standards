# 需求助手

> **语言**: [English](../../../../../skills/claude-code/requirement-assistant/SKILL.md) | 繁体中文

**版本**: 1.0.0
**最后更新**: 2025-12-24
**适用范围**: Claude Code Skills

---

## 目的

本 Skill 提供撰写清晰、完整且可执行需求的指导。

## 快速参考

### 使用者故事格式（INVEST）

```
作为 [角色]，
我想要 [功能]，
以便 [效益]。
```

### INVEST 原则

| 原则 | 说明 | 应该问的问题 |
|------|------|--------------|
| **I**ndependent（独立性） | 可以单独交付 | 这是否依赖其他故事？ |
| **N**egotiable（可协商性） | 细节可以讨论 | 这是否太过具体规定？ |
| **V**aluable（有价值性） | 提供使用者价值 | 这解决了什么问题？ |
| **E**stimable（可估算性） | 可以估算工作量 | 我们了解范围吗？ |
| **S**mall（小型化） | 适合在一个冲刺内完成 | 我们可以拆分这个吗？ |
| **T**estable（可测试性） | 有明确的验收标准 | 我们如何知道完成了？ |

### 需求优先级

| 优先级 | 标签 | 说明 |
|--------|------|------|
| P0 | 必须有 | 发布的关键需求 |
| P1 | 应该有 | 重要但不阻塞 |
| P2 | 可以有 | 可有可无 |
| P3 | 不会有 | 超出范围（本次发布） |

## 快速范本

### 简单 Issue 范本

```markdown
## 问题
[我们要解决什么问题？]

## 建议解决方案
[我们应该如何解决？]

## 验收标准
- [ ] [标准 1]
- [ ] [标准 2]
- [ ] [标准 3]
```

### 功能需求范本

```markdown
## 摘要
[一行描述]

## 动机
[为什么需要这个？谁会受益？]

## 详细描述
[功能的完整描述]

## 验收标准
- [ ] [可衡量的标准 1]
- [ ] [可衡量的标准 2]

## 超出范围
- [这个功能不包含什么]
```

### 错误回报范本

```markdown
## 描述
[错误的简要描述]

## 重现步骤
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

## 预期行为
[应该发生什么]

## 实际行为
[实际发生什么]

## 环境
- 作业系统：[例如：Windows 11]
- 版本：[例如：v1.2.3]
```

## 验收标准指南

### 好的验收标准

- **具体的**：清晰、无歧义
- **可衡量的**：可以验证通过/失败
- **可实现的**：技术上可行
- **相关的**：与需求相关
- **可测试的**：可以为其编写测试

### 范例

**好的范例**：
```markdown
- [ ] 使用者可以上传最大 10MB 的档案
- [ ] 系统在第 95 百分位数内 500ms 内回应
- [ ] 上传失败时显示错误讯息
```

**不好的范例**：
```markdown
- [ ] 系统应该很快  # 无法衡量
- [ ] 使其使用者友善  # 太模糊
- [ ] 修复错误        # 没有具体标准
```

## 需求完整性检查清单

撰写需求时，确保涵盖：

- [ ] **什么**：功能的清晰描述
- [ ] **为什么**：商业价值/解决的问题
- [ ] **谁**：目标使用者/角色
- [ ] **何时**：优先级/时间表
- [ ] **如何**：高阶方法（如果已知）
- [ ] **验收**：完成标准
- [ ] **范围**：不包含什么

## 设定检测

此 Skill 支援专案特定的需求范本。

### 检测顺序

1. 检查 `CONTRIBUTING.md` 中的「停用的 Skills」区段
   - 如果此 Skill 在列表中，则此专案停用此 Skill
2. 检查 `CONTRIBUTING.md` 中的「需求语言」区段
3. 检查 `.github/ISSUE_TEMPLATE/` 目录
4. 检查 `docs/templates/` 目录
5. 如果未找到，**预设使用英文**并使用预设范本

### 首次设定

如果未找到范本：

1. 询问使用者：「此专案没有需求范本。我应该使用哪种语言？（English / 中文）」
2. 使用者选择后，建议在 `CONTRIBUTING.md` 中记录：

```markdown
## 需求语言

此专案使用 **[选择的选项]** 撰写需求和 issues。
<!-- 选项：English | 中文 -->
```

3. 根据专案类型建议适当的范本

### 设定范例

在专案的 `CONTRIBUTING.md` 中：

```markdown
## 需求语言

此专案使用 **English** 撰写需求和 issues。
<!-- 选项：English | 中文 -->

### Issue 范本位置
`.github/ISSUE_TEMPLATE/`
```
