# 腦力激盪助手

> **語言**: [English](../../../../skills/brainstorm-assistant/SKILL.md) | 繁體中文

在撰寫規格前進行結構化發想。透過引導式腦力激盪，將模糊構想轉化為可執行的功能提案。

## 工作流程

```
FRAME ──► DIVERGE ──► CONVERGE ──► OUTPUT
定義問題     發散思考       收斂評估       輸出提案
```

### 階段 1：FRAME | 定義問題

在產生想法之前，先清楚定義問題空間。

| 步驟 | 動作 |
|------|------|
| 1 | 用 5 Whys 釐清問題根因 |
| 2 | 重構為「How Might We」(HMW) 問題 |
| 3 | 識別利害關係人與限制條件 |
| 4 | 從程式碼庫蒐集脈絡（如適用） |

### 階段 2：DIVERGE | 發散思考

不加評判地盡可能產生多個想法。

| 技法 | 使用時機 |
|------|----------|
| **HMW 問題** | 預設起點 |
| **SCAMPER** | 改善現有功能 |
| **六頂思考帽** | 需要多角度思考 |

### 階段 3：CONVERGE | 收斂評估

使用結構化標準評估與排序想法。

| 評估標準 | 權重 |
|----------|------|
| 技術可行性 | 30% |
| 使用者影響力 | 30% |
| 實作成本 | 20% |
| 目標一致性 | 20% |

### 階段 4：OUTPUT | 輸出提案

產生可直接對接 `/requirement` 或 `/sdd` 的腦力激盪報告。

## 技法速覽

| 技法 | 用途 | 步驟 |
|------|------|------|
| **5 Whys** | 根因分析 | 連問 5 次「為什麼？」 |
| **HMW** | 問題重構 | 「我們如何能 [動詞] [成果]？」 |
| **SCAMPER** | 創意改造 | 7 步驟：替代、結合、調適、修改、另作他用、刪除、反轉 |
| **六頂思考帽** | 多角度思考 | 6 種模式：事實、情感、風險、好處、創意、流程 |
| **點數投票** | 快速排序 | 每人 3 票，投給最看好的想法 |

## 輸出格式

```markdown
# 腦力激盪報告：[主題]

## 問題陳述
[FRAME 階段精煉的問題]

## HMW 問題
1. 我們如何能...？
2. 我們如何能...？
3. 我們如何能...？

## 產生的想法
| # | 想法 | 來源技法 | 可行性 | 影響力 | 分數 |
|---|------|----------|--------|--------|------|
| 1 | ...  | SCAMPER   | 4/5    | 5/5    | 4.3  |
| 2 | ...  | HMW       | 3/5    | 4/5    | 3.5  |

## 前 3 名推薦
1. **[想法名稱]** — [推薦原因]
2. **[想法名稱]** — [推薦原因]
3. **[想法名稱]** — [推薦原因]

## 後續步驟
- [ ] 以首選想法進入 `/requirement`
- [ ] 若需求已明確，直接進入 `/sdd`
- [ ] 需進一步探索想法 #N
```

## 使用方式

- `/brainstorm` — 啟動互動式腦力激盪
- `/brainstorm "用戶留存"` — 針對特定主題進行腦力激盪
- `/brainstorm --technique scamper` — 使用特定技法
