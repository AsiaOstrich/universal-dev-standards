# 知識傳承標準

> 版本: 1.0.0 | 最後更新: 2026-04-01

## 概述

本文件定義知識傳承標準，涵蓋 Onboarding、Handoff、Bus Factor 評估及 Code Tour 機制。目標是確保關鍵知識不會因團隊成員離職或角色轉換而消失。

---

## Onboarding 路線圖

### 4 週計畫

| 週次 | 主題 | 目標 | 交付物 |
|------|------|------|--------|
| **第 1 週** | 環境與文化 | 完成開發環境設定；理解團隊規範 | 成功建置並執行專案 |
| **第 2 週** | 架構與程式碼 | 理解系統架構與關鍵路徑 | 完成 Code Tour |
| **第 3 週** | 流程與工具 | 理解 CI/CD、部署與監控流程 | 完成第一個 PR |
| **第 4 週** | 獨立貢獻 | 獨立完成一個小型任務 | 通過 Buddy 評估 |

### Onboarding 檢查清單

| 類別 | 項目 |
|------|------|
| **帳號存取** | Git 儲存庫、CI/CD 平台、監控工具、雲端控制台、通訊工具 |
| **環境設定** | 開發環境、IDE 配置、VPN、資料庫存取、本地建置驗證 |
| **文件** | README、架構文件、API 文件、CLAUDE.md 或等效 AI 脈絡檔案 |
| **會議** | 與 Buddy 1:1、團隊介紹、架構概覽 |
| **首要任務** | Good First Issue、Code Tour、第一個小型 PR |

## Handoff 流程

| 項目 | 說明 | 時程 |
|------|------|------|
| **負責清單** | 列出所有負責的模組、服務與流程 | 離職前 4 週 |
| **隱性知識記錄** | 記錄所有「只有我知道」的操作與陷阱 | 離職前 3 週 |
| **WIP 交接** | 移交進行中的任務、開啟的 PR、事件追蹤 | 離職前 2 週 |
| **Buddy 配對** | 指定繼任者並進行 Pair Programming | 離職前 2 週 |
| **帳號與權限** | 移交 owner 權限、輪換 API 金鑰 | 離職前 1 週 |
| **Runbook 更新** | 審查並更新所有相關 Runbook | 離職前 1 週 |

### 知識記錄格式

| 欄位 | 說明 |
|------|------|
| **主題** | 捕捉什麼知識 |
| **情境** | 何時需要此知識 |
| **步驟** | 具體操作程序 |
| **陷阱** | 常見錯誤與注意事項 |
| **資源** | 程式碼、文件、Runbook 連結 |

## Bus Factor 評估

| 指標 | 定義 | 風險等級 |
|------|------|---------|
| **知識人數** | 可獨立處理該領域的人數 | 1 人 = **高**、2 人 = **中**、3+ 人 = **低** |
| **文件覆蓋率** | 有書面文件覆蓋的百分比 | < 30% = **高**、30-70% = **中**、> 70% = **低** |
| **最近知識分享** | 最近一次有人學習該領域 | > 6 個月 = **高**、3-6 個月 = **中**、< 3 個月 = **低** |

### 知識擴散策略

| 策略 | 適用場景 |
|------|---------|
| **Pair Programming** | 複雜邏輯、特殊技巧 |
| **Tech Talk** | 架構決策、設計理由 |
| **文件化** | 操作程序、部署步驟 |
| **Code Review 輪調** | 程式碼理解、打破知識孤島 |
| **On-call 輪調** | 維運知識、事件回應 |

## Code Tour

### 導覽路線

| 路線 | 涵蓋內容 | 目標對象 |
|------|---------|---------|
| **Quick Start** | 進入點、主要設定、啟動流程 | 所有新人 |
| **Request Flow** | 請求從進入到回應的完整路徑 | 後端開發者 |
| **Data Flow** | 資料從建立到儲存到擷取的路徑 | 資料相關開發者 |
| **Deploy Flow** | 從 PR 到生產的完整路徑 | 需要部署的人 |
| **Key Decisions** | 重要架構決策及其 ADR | 需要理解「為什麼」的人 |
