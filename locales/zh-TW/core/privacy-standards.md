# 隱私標準

> 版本: 1.0.0 | 最後更新: 2026-04-01

## 概述

本文件定義在軟體生命週期中保護使用者資料的隱私標準，基於 Privacy by Design 原則與國際資料保護法規。

## Privacy by Design 七大原則

| # | 原則 | 說明 |
|---|------|------|
| 1 | **主動而非被動** | 在隱私問題發生前預防 |
| 2 | **預設隱私保護** | 個人資料自動受到保護 |
| 3 | **隱私嵌入設計** | 隱私是系統架構的一部分，非附加元件 |
| 4 | **完整功能 — 正和** | 隱私與功能兼得，非二擇一 |
| 5 | **端到端安全** | 資料在整個生命週期中受到保護 |
| 6 | **可見性與透明度** | 所有實踐可文件化、可驗證 |
| 7 | **尊重使用者隱私** | 使用者利益至上，提供強預設與明確選項 |

## 資料分類

| 等級 | 說明 | 處理要求 |
|------|------|---------|
| **公開** | 可自由分享（行銷內容、文件） | 無限制 |
| **內部** | 僅供內部使用（內部報告） | 存取控制 |
| **機密** | 敏感商業資料（財務、策略） | 加密、稽核日誌、需知原則 |
| **受限** | 高度敏感 PII、健康、財務資料 | 最強加密、嚴格存取、資料遮罩 |

## 資料保護影響評估（DPIA）

DPIA 必須在以下情況執行：
- 大規模處理個人資料
- 系統性監控公共區域
- 有法律效力的自動化決策
- 處理特殊類別資料（健康、生物識別）
- 合併不同來源的資料集
- 處理弱勢族群資料（兒童、員工）

## 資料最小化

| 原則 | 說明 |
|------|------|
| **僅收集必要資料** | 每個資料欄位須有文件化的用途 |
| **定義保留期限** | 每種資料類型須有明確保留期限 |
| **到期自動刪除** | 超過保留期限的資料須自動刪除或匿名化 |
| **目的限制** | 為某目的收集的資料不得未經同意用於他途 |

### 保留期限指南

| 資料類型 | 建議保留期限 |
|---------|------------|
| Session 資料 | Session 期間 |
| 活動日誌 | 90 天 |
| 帳號資料 | 帳號刪除 + 30 天 |
| 交易記錄 | 7 年 |
| 行銷偏好 | 直到撤回同意 |

## 使用者權利

| 權利 | 說明 | 實作方式 |
|------|------|---------|
| **存取** | 請求個人資料副本 | 匯出端點（JSON/CSV） |
| **更正** | 修正不正確的資料 | 編輯個人檔案 |
| **刪除** | 請求刪除資料 | 帳號刪除流程 |
| **可攜** | 以可攜格式接收資料 | 標準格式匯出 |
| **反對** | 反對特定處理活動 | Opt-out 機制 |

## 隱私檢查清單

- [ ] 所有個人資料欄位已文件化用途與法律依據
- [ ] 僅收集必要資料
- [ ] 已取得使用者同意（opt-in）
- [ ] 保留期限已定義且自動刪除已設定
- [ ] 僅授權人員可存取
- [ ] 資料於靜止與傳輸中加密
- [ ] 使用者可存取、更正、刪除與匯出其資料
- [ ] 與第三方的資料共享已文件化
- [ ] DPIA 已完成（如符合觸發條件）
- [ ] 資料外洩通知流程已就位
