# 估算標準

> 版本: 1.0.0 | 最後更新: 2026-04-01

## 概述

本文件定義軟體開發專案的估算標準，涵蓋估算方法、校準機制、常見反模式、信心等級、重新估算觸發條件，以及估算與承諾的關鍵區別。

好的估算不在於精確，而在於對不確定性保持透明，並透過校準持續改進。

## 估算方法

### 1. Planning Poker

共識式估算技術，團隊成員獨立使用編號卡片估算，然後討論收斂。

| 面向 | 說明 |
|------|------|
| **適用場景** | Sprint 規劃、Story 級別估算 |
| **團隊規模** | 3–9 人 |
| **尺度** | Fibonacci（1, 2, 3, 5, 8, 13, 21） |

### 2. T-Shirt Sizing

使用抽象尺寸標籤（XS, S, M, L, XL）的相對大小估算技術。

| 面向 | 說明 |
|------|------|
| **適用場景** | Roadmap 規劃、Epic 級別估算 |
| **尺度** | XS, S, M, L, XL |

### 3. 三點估算

使用樂觀值（O）、最可能值（M）、悲觀值（P）的統計技術。

| 面向 | 說明 |
|------|------|
| **適用場景** | 關鍵路徑項目、高不確定性任務 |
| **公式** | E = (O + 4M + P) / 6; SD = (P - O) / 6 |

## 校準

團隊應定期將估算與實際結果比對，以持續提升準確度。

**估算準確率**：`實際工作量 / 估算工作量`
- 比率 = 1.0：完美估算
- 比率 > 1.0：低估（實際花更久）
- 比率 < 1.0：高估（實際花更少）

## 反模式

| 反模式 | 說明 | 緩解策略 |
|--------|------|---------|
| **錨定偏誤** | 第一個數字主導後續估算 | 使用盲估（如同時翻牌） |
| **規劃謬誤** | 系統性低估時間和風險 | 使用歷史校準數據 |
| **範圍蔓延盲區** | 未注意逐漸擴大的範圍 | 估算前鎖定範圍定義 |
| **學生症候群** | 拖到最後才開始工作 | 設定中間里程碑 |
| **帕金森定律** | 工作擴展以填滿可用時間 | 使用相對估算（Story Points） |

## 信心等級

| 等級 | 變異範圍 | 適用情境 |
|------|---------|---------|
| **高** | ±20% | 需求明確、有先前經驗的熟悉工作 |
| **中** | ±50% | 部分已知、有些未知的工作 |
| **低** | ±100% | 高度不確定、新技術或不明確需求 |

## 重新估算觸發條件

| 觸發條件 | 行動 |
|---------|------|
| **需求變更** | 重新估算受影響項目 |
| **技術發現** | 以更新的知識重新估算 |
| **外部相依變更** | 重新估算整合工作 |
| **時間門檻超過** | 進行檢查點審查 |

## 估算 vs 承諾

| 面向 | 估算 | 承諾 |
|------|------|------|
| **本質** | 帶有不確定性的預測 | 帶有責任的承諾 |
| **負責人** | 開發團隊 | 管理層 / Product Owner |
| **格式** | 範圍 + 信心等級 | 具體日期或範圍 |
| **變更** | 隨資訊更新而調整 | 與利害關係人重新協商 |
