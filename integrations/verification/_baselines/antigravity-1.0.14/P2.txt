### 2. How Migrations Are Run
In a Prisma setup, database migrations are managed using the Prisma CLI (`npx prisma`):

- **Development environment**:
  ```bash
  npx prisma migrate dev --name <migration_name>
  ```
  *(Creates a new SQL migration file from changes in `schema.prisma` and applies it to the database)*

- **Production / CI/CD environment**:
  ```bash
  npx prisma migrate deploy
  ```
  *(Applies pending migration files to the database)*

- **Prototyping / Direct sync**:
  ```bash
  npx prisma db push
  ```
  *(Pushes the schema state directly to the database without creating migration files)*
