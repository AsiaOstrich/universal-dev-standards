## Why separate

Different tool, different models, different loading mechanism. A survival rate measured on
Antigravity says nothing about Codex and vice versa — the whole point of running both is that
they can disagree. Keeping one table per tool is what makes a disagreement visible instead of
being averaged away.

Antigravity is also the only vehicle here that can vary the **model** while holding the tool
constant: `agy --model` offers 11 models across Gemini, Claude and GPT-OSS. That isolates model
effects from tool effects, which no single-model tool can do.

## What has NOT been done here

**Outcome measurement.** Everything above measures *form* — whether the declared vocabulary
appears. The blind-judged outcome method (does the review actually get better) has only been
run against Codex.

Running it here would be a **new experiment with its own data set**, not an extension of the
Codex one. It needs: the same subject file, reviews generated by `agy`, and the same two-judge
blind protocol. The judges themselves are already Antigravity models, so a third-party judge
would be needed to keep the judge separate from the tool under test.

## Also serving as judge

Antigravity models judged the Codex outcome runs (Gemini 3.1 Pro and Claude Sonnet 4.6). That
is legitimate — the judge is not the tool under test — but it means **Antigravity cannot judge
its own outcome runs**. Codex would be the natural judge for those, and Codex is currently
quota-blocked until 2026-08-22.
