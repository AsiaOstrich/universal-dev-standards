### 2. How Migrations Are Run
Since Prisma is configured, migrations are run via the Prisma CLI:

- **In Development** (create and apply migration files):
  ```bash
  npx prisma migrate dev --name <migration_name>
  ```
- **In Production / Staging** (apply existing pending migrations):
  ```bash
  npx prisma migrate deploy
  ```
- **Direct Schema Push** (prototyping without generating migration files):
  ```bash
  npx prisma db push
  ```

If you haven't initialized Prisma yet, run `npx prisma init` to generate the default `prisma/schema.prisma` and `.env` files.
