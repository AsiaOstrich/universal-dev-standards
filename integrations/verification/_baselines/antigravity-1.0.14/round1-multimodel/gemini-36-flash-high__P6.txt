### 1. Plaintext Password Logging (Security Risk)
* **Location:** [auth.js:L2](file:///private/tmp/claude-502/-Users-alberthsu-Documents-GitHub-AsiaOstrich-dev-platform/c517e347-06b6-4833-abc4-32d57356c1b9/scratchpad/cr-test/src/auth.js#L2)
* **Issue:** `console.log("login attempt", user, pass);` logs the plain-text password `pass` and user details into application logs.
* **Fix:** Avoid logging sensitive authentication credentials.

### 2. Hardcoded Secret / API Key (Security Risk)
* **Location:** [auth.js:L3](file:///private/tmp/claude-502/-Users-alberthsu-Documents-GitHub-AsiaOstrich-dev-platform/c517e347-06b6-4833-abc4-32d57356c1b9/scratchpad/cr-test/src/auth.js#L3)
* **Issue:** `const KEY = "sk-live-9f3a2b8c1d";` hardcodes what appears to be an active API key into the source code.
* **Fix:** Store secrets in environment variables (e.g., `process.env`) and remove unused declarations.

### 3. Loose Equality & Plaintext Password Comparison (Security & Integrity Risk)
* **Location:** [auth.js:L4](file:///private/tmp/claude-502/-Users-alberthsu-Documents-GitHub-AsiaOstrich-dev-platform/c517e347-06b6-4833-abc4-32d57356c1b9/scratchpad/cr-test/src/auth.js#L4)
* **Issue:** 
  1. `pass == user.password` uses loose equality (`==`), which allows unexpected type coercion (e.g., `0 == ""` evaluates to `true`).
  2. Plain-text comparison is vulnerable to timing attacks and indicates passwords may not be properly hashed.
* **Fix:** Use secure, constant-time comparison methods (or password hashing libraries like `bcrypt` or `argon2`) and strict equality (`===`).

### 4. Unhandled Null / Undefined Objects (Runtime Error Risk)
* **Location:** [auth.js:L4](file:///private/tmp/claude-502/-Users-alberthsu-Documents-GitHub-AsiaOstrich-dev-platform/c517e347-06b6-4833-abc4-32d57356c1b9/scratchpad/cr-test/src/auth.js#L4)
* **Issue:** `user.password` will throw a `TypeError` if `user` is `null` or `undefined`.
* **Fix:** Add parameter validation or optional chaining (`user?.password`).
