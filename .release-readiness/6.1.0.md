## What's in this release

**XSPEC-340 — `verification-evidence` 1.1.0 → 1.2.0: evidence validity.**
The standard treated `exit_code` as ground truth. It had already anticipated that an *agent* can report a false success; it never considered that a *tool* can. On 2026-07-17 that contract was falsified ten times in one day — four commands exited 0 and were false, two exited non-zero against healthy artefacts, and following VE-002 on those two deleted a good encrypted backup twice. New `evidence_validity` layer + VE-007–VE-010; `trust_rules`, VE-002 and `physical_spec.checks` amended in place rather than contradicted from a new section. Also backfills v1.1.0's `environment_layer` into `core/*.md`, where it had been missing since 2026-05-13, and promotes three sections that existed only in the zh-TW translation.

**XSPEC-341 — `uds init` no longer overwrites an existing `prepare` script.**
Since 2026-02-04, `uds init` ran `npx husky init` on any Node project without `.husky/`. That command sets `"prepare": "husky"` unconditionally. For a published package whose `prepare` is its build step, the next `npm publish` would ship an unbuilt `dist/` — and the CLI printed success. `setupHuskyHook` had no test; it now has 10, running against real temp directories, and RED-GREEN holds (4 fail pre-fix, 10 pass post-fix).

**Also**: `requirement-engineering` v1.1.0 (checklist generation) + `/sdd` convergence check; `frontend-design-standards` v1.1.0 (token interpolation); layered test-scope exception for checkin/push gates (XSPEC-326 R4); `/brainstorm` v4.1 score-threshold recommendations; cross-conversation language consistency checks; `source_hash` translation drift detection + `refresh-translation-hash` tool; bundle-parity gate wired into the bump scripts; ~36 commits of translation-debt paydown and stale-count fixes.
