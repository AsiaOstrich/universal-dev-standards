## What's in this release

**XSPEC-342 R1 — `uds check` compares standards to npm latest, not the CLI's bundled copy.**
`displayAdoptionStatus` checked `manifest.upstream.version` against the standards bundled *inside the running CLI* (`repoInfo.standards.version`). When the CLI itself was stale, that copy was older than npm — producing a backwards `⚠ Update available: 6.1.0 → 5.12.1` and structurally never able to say your standards were behind. This live bug was reproduced against vibeops (installed 6.1.0, an old global CLI bundling 5.12.1) before the fix. Now it feeds on `checkForUpdates(manifest.upstream.version)`: `available` = installed < npm latest. Message reframed (EN/zh-TW/zh-CN) to "your installed standards are behind" with the full two-step fix (`npm update -g` **and then** `uds update`); `--offline` skips silently.

**XSPEC-342 R4 (quiet-pass subset) — `uds check` no longer lists every unchanged file.**
It printed one `✓ … (unchanged)` line per tracked file — ~70% of the command's output (measured 121 → 41 lines) — burying the messages that needed reading and making the output large enough that pre-commit agents truncated it. Per-file "unchanged" listing removed; the count stays in the one-line integrity summary; modified/missing/unhashed files still listed. (Broader R4 — other pre-commit gates, agent-truncation norm — remains out of scope.)
